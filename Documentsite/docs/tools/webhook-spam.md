# Webhook Spamの使い方

## 概要
Webhook Spamは、Discordサーバー内にWebhookを作成し、大量のメッセージを自動送信するツールです。通常のメッセージ送信とは異なり、Webhookを使用することでレート制限を回避しながら効率的な荒らしを実行できます。

## アクセス
ツールURL: https://lookjbn.github.io/ozeu-webhook-spam/

## UI構成と使い方

### 基本設定

#### トークン設定
```
トークン 
Webhook作成用のトークン
```
- Webhook作成権限を持つDiscordトークンを入力
- サーバー管理権限またはWebhook管理権限が必要

#### ターゲット設定
```
サーバーID 
1362131922347622462
```
- Webhookを作成するサーバーのIDを指定
- 対象サーバーでの適切な権限が必要

### メッセージ設定

#### Webhook設定
```
Webhook名 
Captain Hook
```
- 作成するWebhookの表示名を設定
- 任意の名前を指定可能

#### 送信内容
```
メッセージ内容 
@everyone
```
- Webhookが送信するメッセージ内容
- メンション、絵文字、埋め込みコンテンツに対応

### 実行制御
- **実行**: Webhookスパムを開始
- **停止**: 実行中のスパムを緊急停止

## 実行手順

### 基本操作
1. **トークン入力**: Webhook作成権限を持つトークンを設定
2. **サーバー指定**: ターゲットサーバーのIDを入力
3. **Webhook設定**: 作成するWebhookの名前を指定
4. **メッセージ設定**: 送信するメッセージ内容を入力
5. **実行開始**: 「実行」ボタンでスパム開始
6. **停止操作**: 必要に応じて「停止」ボタンで中断

### 権限確認
- サーバー管理権限の確認
- Webhook管理権限の確認
- チャンネルへのアクセス権限の確認

## 荒らし戦術

### 基本的な攻撃パターン

#### 全体メンション攻撃
```
メッセージ内容: @everyone 重要なお知らせ
効果: 全メンバーに通知を強制送信
```

#### ロール指定攻撃
```
メッセージ内容: @here @役職名 緊急連絡
効果: 特定役職への集中攻撃
```

#### 連続メッセージ攻撃
```
メッセージ内容: スパムメッセージ×大量
効果: チャンネルを完全に埋め尽くす
```

### 高度な攻撃手法

#### 複数Webhook同時実行
- 異なる名前で複数Webhookを作成
- 同時並行でメッセージスパム実行
- サーバー全体への負荷集中攻撃

#### チャンネル横断攻撃
- 複数チャンネルにWebhookを配置
- サーバー全体でのメッセージ氾濫
- 管理者の対応を困難にする

### Webhookの利点

#### レート制限回避
- 通常のメッセージ送信より制限が緩い
- 大量メッセージの高速送信が可能
- ユーザーアカウントの制限を迂回

#### 匿名性
- Webhook名でメッセージ送信者を偽装
- 実際のユーザー名を隠蔽
- 追跡を困難にする効果

## 効果的な運用方法

### 事前準備
1. **権限確認**: 必要な管理権限の取得
2. **ターゲット調査**: サーバー構造とチャンネル配置の把握
3. **Webhook戦略**: 配置場所と実行タイミングの計画

### 実行戦略
- **時間帯選択**: アクティブユーザーが多い時間を狙う
- **メッセージ内容**: 最大限の混乱を誘発する内容
- **継続時間**: 管理者の対応前に最大効果を得る

### 対策回避
- **Webhook名変更**: 発見を遅らせるため定期的に変更
- **メッセージ変化**: パターン認識による自動削除を回避
- **分散実行**: 複数の時間帯に分けて実行

## トラブルシューティング

### よくあるエラー
- **権限不足**: Webhook作成権限の確認
- **レート制限**: 実行間隔の調整
- **サーバーID誤り**: 正しいサーバーIDの確認

### 対処方法
- エラーメッセージの確認
- 権限設定の再確認
- 別のトークンでの実行試行

## 注意点

### 実行前確認
- 対象サーバーでの権限状況
- Webhook作成の制限事項
- メッセージ内容の適切性

### 効率的な荒らし
- 他の荒らしツールとの連携
- タイミングの最適化
- 発見回避のための工夫
